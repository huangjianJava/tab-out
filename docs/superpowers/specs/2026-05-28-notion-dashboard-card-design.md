# Tab Out Notion Database Card 视觉重塑设计

## 背景

Tab Out 当前是一个纯 Chrome 扩展，新标签页会展示所有已打开标签页，并按域名分组。现有页面使用偏复古纸张感的暖色调、serif 标题字体、暖橙/鼠尾草绿/玫瑰色强调色，以及卡片顶部状态色条。

这次调整的目标是把页面改成更接近 Notion 的清爽专业风格，同时保留 Tab Out 的核心效率：一眼看清、快速跳转、快速关闭、保存稍后再看。

## 设计目标

- 页面整体更像 Notion database gallery：白底卡片、细边框、轻量标签、安静的工作台感。
- 配色从暖纸色和复古橙棕转为白、浅灰、墨黑，少量蓝绿作为功能强调。
- 保留现有信息架构和交互，不改变 Chrome API 行为、数据结构或功能范围。
- 提升标签页扫描效率，让域名分组、重复项、主页分组、稍后再看更加清晰。
- 视觉更耐看，适合作为每天频繁打开的新标签页。

## 范围

本次只做视觉重塑：

- 更新 `extension/style.css` 的视觉变量、布局细节和组件样式。
- 必要时微调 `extension/index.html` 中字体引用，移除不再需要的外部字体。
- 如需同步 confetti 颜色，可在 `extension/app.js` 中只调整颜色数组。

不做以下事情：

- 不重写标签页分组逻辑。
- 不改变 saved for later 的存储结构。
- 不新增设置页、主题切换或复杂自定义能力。
- 不改变关闭、跳转、去重、归档等既有行为。

## 视觉系统

### 字体

使用系统字体栈，接近 Notion 的默认观感：

- `-apple-system`
- `BlinkMacSystemFont`
- `Segoe UI`
- `Inter`
- `Helvetica Neue`
- `Arial`
- `sans-serif`

移除当前 `Newsreader` serif 标题感，让问候语、分区标题、数字和卡片标题都更克制。

### 色彩

基础颜色：

- 页面背景：近白浅灰，例如 `#f7f7f5`
- 卡片背景：白色或近白，例如 `#ffffff` / `#fbfbfa`
- 主文字：墨黑，例如 `#202020`
- 次级文字：中性灰，例如 `#6f6f6f`
- 边框：浅灰，例如 `#e6e6e3`
- hover 背景：极浅灰，例如 `#f1f1ef`

强调色只用于表达状态，不做大面积装饰：

- 主页分组：淡蓝
- 重复项：淡琥珀
- 大分组或清理建议：淡绿
- 危险或关闭 hover：低饱和红
- 普通状态：灰色

### 形状和阴影

- 卡片圆角保持克制，建议 `6px` 或 `8px`。
- 默认卡片不使用明显阴影，以边框区分层级。
- hover 可使用非常轻的阴影和浅灰背景，避免漂浮感太强。
- 去掉卡片顶部彩色状态条，改由 badge 和文字状态表达优先级。

## 页面结构

保留现有结构：

- 顶部 header：问候语和日期。
- 主区域：打开的标签页分组网格。
- 右侧栏：稍后再看清单。
- 底部：打开标签页统计。

视觉调整：

- header 间距更紧凑，底部分割线更淡。
- 主容器仍保持宽屏 dashboard，但整体更像一个 Notion workspace。
- 左侧网格继续使用当前 masonry/card 布局，保证多标签页场景下的扫描效率。
- 右侧“稍后再看”保持 sidebar 感，但样式改成 checklist database。

## 组件设计

### 顶部区域

问候语使用中等字号和 medium 字重，不再使用 serif 或 italic。日期使用小号灰色文本，保持安静。

### 健康摘要

`tab-health-summary` 改成 Notion callout/database summary 风格：

- 白底或浅灰底。
- 细边框。
- 指标块改成轻量 inline stats。
- 不使用渐变背景。

### 域名卡片

`mission-card` 是本次视觉重塑的核心：

- 白底。
- 1px 浅灰边框。
- `6px` 或 `8px` 圆角。
- 卡片内边距略收紧。
- 标题区像 Notion database card 的属性区。
- 不再显示彩色顶部条。
- hover 时边框略深、背景略灰，或出现极轻阴影。

### 标签页行

`page-chip` 改成 Notion page row：

- 保留 favicon、标题、保存按钮、关闭按钮。
- 行间分割线更淡。
- hover 使用浅灰底。
- 标题最多两行，继续避免过长标题撑破卡片。
- 操作按钮默认低调，hover 时清晰显示。

### Badge

所有 `priority-badge` 改为 Notion 式属性标签：

- 小字号。
- 浅色背景。
- 低饱和文字。
- 不使用强烈 uppercase 感。
- 重复项、主页、大分组、普通状态分别使用不同浅色。

### 操作按钮

`action-btn`、`tab-cleanup-btn`、archive 按钮等改成 Notion 小按钮：

- 白底或浅灰底。
- 细边框。
- 黑/灰文字。
- hover 使用浅灰背景。
- 主要操作可用深色文字或轻微蓝绿强调，但不使用大面积亮色按钮。

### 稍后再看

右侧栏改成轻量 checklist database：

- 条目像 Notion checklist row。
- 完成、恢复、删除等操作保持低调。
- 空状态文字保持简短，但视觉上更像 Notion 的空 database。

### Toast 与动画

toast 改成 Notion 风格的浮层：

- 白底。
- 细边框。
- 轻阴影。
- 简洁图标与文字。

swoosh 音效保留。confetti 保留，但颜色改为更克制的蓝、绿、灰、淡琥珀，弱化当前暖橙复古感。

## 响应式要求

- 桌面宽屏继续展示打开标签页网格和稍后再看侧栏。
- 中等宽度下保持卡片列数自适应。
- 小屏下侧栏自然下移，不产生横向滚动。
- 按钮、badge、长标题不得互相遮挡。
- 卡片 hover 和操作按钮出现时不得造成布局跳动。

## 验证标准

实现完成后需要检查：

- 新标签页整体观感明确接近 Notion database card，而不是复古纸张风格。
- 页面主色调为白、浅灰、墨黑，强调色克制。
- 打开标签页很多时，分组仍然容易扫描。
- 重复项、主页分组、稍后再看仍然容易识别。
- 关闭标签页、保存稍后再看、清理重复项、跳转标签页行为不变。
- 桌面和窄屏视口下文本不溢出、不重叠。

## 实施建议

优先从 CSS 变量开始重塑视觉系统，再依次调整 header、summary、cards、page rows、sidebar、toast 和 confetti。这样可以保持改动集中，并降低影响功能逻辑的风险。
